Read the README.md file within this skill, and explore the other available files.
If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out and create static HTML files for the user to view. If working on production code, you can copy assets and read the rules here to become an expert in designing with this brand.
If the user invokes this skill without any other guidance, ask them what they want to build or design, ask some questions, and act as an expert designer who outputs HTML artifacts _or_ production code, depending on the need.

Key facts to keep in mind:
- One accent color only: raspberry red `#FF3052`. Never introduce a second brand hue; derive hover/active from the same red.
- Two type families with strict roles: **Heavitas** (substituted with Fredoka here — see README Known Gaps) for H1/hero only; **Montserrat** (400/700 only) for everything else.
- Completely flat: no shadows, minimal radii (0/3/5/200/9999px), depth from surface contrast (white / #F8F8F8 / #212121) and photo overlays only.
- Buttons always trail their chevron icon after the label.
- Copy is warm, confident, "we/you" Spanish-language agency voice — see README Content Fundamentals for tone and examples.
